# Seedream 4.5 专项适配设计

## 目标

把 `doubao-seedream-4-5-251128` 从“仅接入基础调用”提升为 PairForge 的正式特化模型，并确保尺寸选择、保存校验和请求参数符合火山方舟当前官方文档。

## 官方依据

- 图片生成 API：<https://www.volcengine.com/docs/82379/1541523>
- Seedream 4.0–5.0 提示词指南：<https://www.volcengine.com/docs/82379/1829186>
- 火山方舟批量推理算子说明：<https://www.volcengine.com/docs/6492/2221472>

截至 2026-07-24，官方文档确认：

- 模型 ID 为 `doubao-seedream-4-5-251128`。
- 普通图片生成使用 `/api/v3/images/generations`；PairForge 现有火山方舟适配器已经使用该路径。
- 支持文生图、参考图生图、图片编辑和组图输出。
- `size` 可使用分辨率档位或显式宽高，两种方式不可混用。
- 4.5 的分辨率档位仅支持 `2K`、`4K`。
- 显式尺寸默认 `2048x2048`，总像素范围为 3,686,400–16,777,216，宽高比范围为 1:16–16:1。
- 2K 常用映射为：1:1 `2048x2048`、4:3 `2304x1728`、3:4 `1728x2304`、16:9 `2848x1600`、9:16 `1600x2848`、3:2 `2496x1664`、2:3 `1664x2496`、21:9 `3136x1344`。
- 单图模式使用 `sequential_image_generation=disabled`；组图模式使用 `auto` 和 `max_images`。现有适配器已经正确实现。

## 方案

### 1. 模型能力

- 将 4.5 默认尺寸由旧的通用横图 `2048x1536` 改为官方默认 `2048x2048`。
- 将 4.5 标记为“已特化”，保留 4.0 的“测试适配”状态。
- 继续共用火山方舟请求适配器，不建立重复的提供商实现。

### 2. 尺寸规则

- 抽象 Seedream 模型级尺寸规格，分别声明允许的分辨率档位、最小/最大总像素和兼容迁移表。
- 4.5 仅接受 `2K`、`4K` 或满足官方像素范围与宽高比的显式尺寸；误填 `3K` 时在保存设置阶段直接给出中文错误，而不是等到批量生成后失败。
- 旧项目中的 4.5 横竖图尺寸继续可读；四个旧预设自动迁移到当前官方 2K 映射，避免新旧项目出现不同的“官方比例”。
- 不改变阿里云或自定义模型规则。

### 3. 前端交互

- 4.5 单独使用官方 2K 常用比例表，不再沿用通用 Seedream 旧尺寸表。
- 补充 3:2、2:3、21:9 三个官方常用比例。
- 设置页说明 4.5 支持 2K/4K，预设使用官方 2K 映射；自定义尺寸仍需满足总像素和宽高比限制。
- 已特化提示根据当前模型显示，避免仍只写 Seedream 5.0 Lite。

### 4. 文档与版本

- README 将 4.5 移入已特化模型，明确当前仍以 5.0 Lite 和 4.5 为火山方舟生产适配目标。
- 版本升级到 `0.6.0`，体现新增一个正式特化模型。

## 验证

- 后端单元测试覆盖 4.5 的默认尺寸、支持级别、2K/4K、显式像素上下限、比例限制和旧尺寸迁移。
- 前端单元测试覆盖 4.5 全部官方 2K 比例映射。
- 请求适配器测试确认 4.5 仍通过同一官方路径发送正确的单图、组图及参考图字段。
- 运行完整后端测试、前端测试、类型检查与生产构建。
- 重新构建 Windows 可执行程序并运行本地启动冒烟测试。
